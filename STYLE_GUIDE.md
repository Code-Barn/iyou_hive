# Hiver Style Guide

## Overview
This style guide ensures consistency across the Hiver project, aligning with the Byers Brands ecosystem.

## Color Palette
- **Primary:** Honey-Orange (`#FF8C00`)
- **Accent:** Byers Blue (`#0064AA`)
- **Dark Mode:**
  - Background: Charcoal (`#1A1A1A`)
  - Text: White (`#FFFFFF`)
- **Light Mode:**
  - Background: Off-White (`#F5F5F5`)
  - Text: Dark Gray (`#333333`)

## Typography
- **Headings:** Bold, sans-serif (e.g., `font-family: 'Helvetica Neue', sans-serif;`)
- **Body Text:** Regular, sans-serif

## UI Components
### Timeline
- **Line:** 2px solid Honey-Orange (`#FF8C00`)
- **Event Dots:** 12px solid Byers Blue (`#0064AA`)
- **Popups:** White background, Honey-Orange border

### Buttons
- **Primary:** Honey-Orange background, white text
- **Secondary:** Byers Blue background, white text

### Forms
- **Input Fields:** Light gray border, Honey-Orange focus state
- **Labels:** Dark gray text

## Layout
- **Grid System:** 12-column grid for responsiveness
- **Spacing:** Use multiples of 8px for margins and padding

## Icons
- Use simple, bold icons for actions (e.g., upload, edit, delete)
- Color: Honey-Orange for active states, gray for inactive

## Accessibility
- Ensure color contrast meets WCAG standards
- Provide alt text for images and icons
- Support keyboard navigation

## Code Style
- Follow Django and Python best practices
- Use descriptive variable and function names
- Comment complex logic for clarity

## Documentation
- Keep documentation up-to-date in the `docs/` directory
- Use Markdown for readability
