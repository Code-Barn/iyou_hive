## рџ”Ќ 405 Upload Error - FORENSIC AUDIT RESULTS

### Problem Identified
The frontend was calling `POST /api/archive/documents/upload/` but receiving **405 Method Not Allowed**.

### Root Cause
**URL Pattern Ordering Conflict** in `apps/archive/api_urls.py`:

1. The DefaultRouter registered patterns for `ArchiveDocumentViewSet`:
   - `^documents/$` - list/create
   - `^documents/(?P<pk>[^/]+)/$` - retrieve/update/delete

2. The explicit `path('documents/upload/', DocumentUploadView.as_view())` was placed **AFTER** the router include.

3. Django's URL matching matched `/api/archive/documents/upload/` to the router's pattern with `pk='upload'`, NOT to the explicit upload endpoint.

4. Since `upload` is not a valid UUID, the ViewSet returned 405 (method not allowed for detail view).

### Fix Applied
**Reordered URL patterns** in `apps/archive/api_urls.py`:

```python
urlpatterns = [
    # Smart Ingestion endpoint - MUST come BEFORE router include
    # to prevent /upload/ from being matched as a pk in the router
    path('documents/upload/', api_views.DocumentUploadView.as_view(), name='api_document_upload'),
    
    # Gate Logic endpoints - also before router
    path('documents/<uuid:pk>/promote/', api_views.ArchiveDocumentViewSet.as_view({'post': 'promote'}), name='archive-document-promote'),
    path('documents/<uuid:pk>/demote/', api_views.ArchiveDocumentViewSet.as_view({'post': 'demote'}), name='archive-document-demote'),
    path('documents/move_file/', api_views.ArchiveDocumentViewSet.as_view({'post': 'move_file'}), name='archive-document-move-file'),
    path('documents/metadata/<uuid:file_uuid>/', api_views.FileMetadataView.as_view(), name='archive-document-metadata'),
    
    # Router for standard CRUD operations (MUST come AFTER explicit paths)
    path('', include(router.urls)),
    
    # Additional endpoints
    path('directory/', api_views.ArchiveDirectoryView.as_view(), name='archive-directory'),
]
```

### Additional Fixes
1. **Fixed import error** in `apps/archive/api_views.py`:
   - Changed `from rest_framework.decorators import method_decorator` to `from django.utils.decorators import method_decorator`
   - Added `ValidationError` to rest_framework exceptions imports

2. **Verified URL Resolution**:
   ```
   /api/archive/documents/upload/ -> DocumentUploadView (POST allowed) вњ…
   /api/archive/documents/ -> ArchiveDocumentViewSet (GET/POST allowed) вњ…
   ```

### Verification
- вњ… URL now correctly resolves to `DocumentUploadView`
- вњ… POST method is explicitly allowed
- вњ… No competing routes at the same path
- вњ… Trailing slash is included in both frontend call and Django pattern
